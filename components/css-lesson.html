So you’ve built a webpage. Or at least, you think it’s a webpage. It looks very different from what you’re used to seeing on the Internet--there are no fun, bright colors, no sleek interfaces, no gifs of adorable mammals. Hey, those tutorial people said they’d tell you how to build a website, but what you have looks like something from the early stages of the ARPANET. Those liars!

We didn’t lie to you (at least, not yet). We just haven’t gotten to the pretty stuff yet – CSS. HTML was fine for writing out what stuff is in your website, but now we need another way to describe how that stuff should look.

You'll need to get started

The syntax for CSS is a lot simpler than the syntax for HTML. Are you ready for it? Here it is: 

				

					
element {
  property: value;
  property: value;
}

A property is an attribute of an element. Common properties include background-color, height, width, and others. A property’s value affects certain parts of the element, usually in how it is displayed. Values can be a range of different things, like a number, word or color.

That’s it, seriously. Okay, that’s not it it, but you can do quite a bit just by modifying the properties of specific elements.

Let’s change the background color of our page. To do that, type the following into your CSS file:

					

						
body {
  background-color: beige;
}

RGB and HSB and Hex, Oh my!

There are a lot of ways to pick color values. The way we have here is to just use a color that the browser already knows, like red or beige. For a more precise method of picking colors, look into RGB, HSB, and hex values.

There! Our website’s looking better already. By playing with elements’ property and value pairs, we can modify just about anything, from size to color to position.

Blast from the past

But there’s still a problem. As of right now, we don’t have any way of grouping certain elements together in our HTML. Let’s say we wanted to have all of our introductory information have the same background-color. We could try styling each piece at once, like this:

				

					
h1 {
  background-color: firebrick;
}

img {
  background-color: firebrick;
}

p {
  background-color: firebrick;
}

But that doesn’t really give us what we want (and even if it was what we wanted, there would be an easier way to do that). Instead of combining the individual elements, we just have three separately colored blocks. If only there was some way to go in our HTML and put the elements into one larger, over-arching element. Oh, wait there is.

Remember how in the HTML lesson we had you wrap all your introductory information in a <div>? Yeah, those were good times. And now we get to actually put that <div> to use.

Now we can target the containing element in our CSS by modifying the <div> like so:

				

					
div {
  background-color: firebrick;
}

So far in our CSS we've been targeting all the instances of an element, and we still don't have a way to target specific elements. What if we wanted to change the font of the items within the social media list? We need some way of labeling the elements we want to change and then apply rules to those specific elements.

Back in your HTML document, dd a class to your header and paragraph and call it white-text, like this: 

				

					
<h1 class='white-text'>
  // your text
<h1>

<p class='white-text'>
  // your text
<p>

Add an ID to your ordered list and call it ‘gray-text.’ 

				

<ol id='gray-text'>
  // list items
<ol>

Let’s target our classes and IDs. To modify an element based on its class, you need to add a period before the class name in the CSS file. 

				

					
.white-text {
  color: white;
}

To target an ID, simply use a pound sign instead of a period. 

					

						
#gray-text {
  color: gray;
}

Be descriptive

Classes and IDs not only make it easier to style your code, but are also a good opportunity to help with organization and readability as well because it allows people (including you!) to understand the purpose of each HTML element at a glance. Give your classes and IDs descriptive names so you can come back to them more easily when you need to modify them again.

Probably one of the biggest things you’ll be modifying in your CSS is your webpage’s layout. Now that you have a grasp over how to change elements’ styles – such as color and size – you’ll want to be able to dictate where those elements go and how they are positioned in relation to each other. This is where the box model comes in handy. In CSS, each element is essentially surrounded by a box made up of margins, borders and padding. It's a lot to take in at once, so head to this standalone lesson to get acquainted with it.

Okay, what we’ve done so far is nice and dandy, but your website still doesn’t look like a website. Let’s change that. You know what a navbar is, right? That thing at the top or side of a website that gives you quick links to key pages on a website. Let’s make one of those. You know, like a real website.

Navbars might seem complicated, but they’re really just styled lists. Let’s turn our unordered list of social media links into our navbar.

First, let’s make this easier on our future selves by applying a descriptive ID to our unordered list. Let’s call it ‘navbar.’

						

							
<ul id='navbar'>
  // list items
</ul>

Now let’s get rid of those ugly little bullets. In your CSS document, target the <li>s nested in your navbar ID and change the list-style-type to none.

					

						
#navbar li{
  list-style-type: none;
}

Next, we want our links to all be in a row instead of stacked on top of each other. To change that, we’ll affect the display property.

					

						
#navbar li{
  list-style-type: none
  display: inline-block;
}

The links seem a little too close to each other, so let’s adjust the width of their <a> elements. We’re adjusting the <a> tags rather than the <li> because we want the user to be able to click on the link from anywhere inside the box that holds the link. We need the links to have a display value of block in order for the width to apply to them, however.

					

						
#navbar li a{
  display: block;
  width: 120px;
}

Great! Now let’s center the text for the links inside their boxes and provide some vertical padding.

					

						
#navbar li a{
  text-align: center;
  padding-top: 10px;
  padding-bottom: 10px;
}

Something’s still a bit off: the placement of our navbar right now just doesn’t make sense. Navbars are supposed to be at the top of the page, but right now our <ul> is buried. In your HTML file, move the <ul> tag and its contents to the top of the body, above the <div>.

Now when we refresh the page the position of the navbar makes sense, although it still doesn’t look quite right visually. That’s because of the top margin. The navbar has some top margin by default, which is keeping it from going to the top of the page. To fix that, change the display of the navbar to be inline-block and get rid of the top margin.

					

						
#navbar {
  display: inline-block;
  margin-top: 0;
}

We want our navbar to stretch across the whole screen, so we need to set its width to 100%. We also want it to be distinct from the rest of the page, so we need to change its background color as well. I’ll be changing mine to darkslategray, but you can pick whatever value you want (try playing around with RGB or even hex values if you’re feeling adventurous).

					

						
#navbar {
  width: 100%;
  background-color: darkslategray;
}

At this point you might also want to change the color of the links inside your navbar if the default blue is hard to see against your background. Experiment with different font-family, color, and other values and find a combination you like. That’s the fun with styling: you can make it anything you want. Like literally almost anything.

Our navbar is almost complete! But there’s just one more thing that would really seal the deal: user feedback. It would be cool if we could change something on the screen so that whenever a user hovered over a link they could be sure of what they were doing.

Guess what? You can do that with CSS. We’ve already covered a few selectors, but there are so many more useful selectors that can be applied in a variety of ways. One class of selectors is referred to as a “pseudo-class” uses a colon (:) to select an element which has a certain property. For example, if we wanted to change the color of links that had already been visited, we would target a:visited in our CSS. For our navbar, we can respond to users by changing the background color of the links they’re hovering over. We’ll use the a:hover selector for that.

					

						
#navbar li a:hover{
  background-color: black;
}

Try it out! There are more useful selectors just like :hover and :visited, and being familiar with them can make your life a whole lot easier and make your website that much more versatile.

Useful selectors

  1. #element {

    The pound sign allows you to select classes, which is very useful for styling multiple elements at once

  2. .element {

    Like the class selector, a period allows you to select an element based on its unique ID

  3. element:hover {

    The hover pseudo-class will apply rules to an element once it sees a mouse hovering over it, and is extremely useful in a wide range of applications

  4. element[attr='val'] {

    This selector will only select instances of the element which have val as the value for a given attribute. For example, if you wanted to target all links on your page that went to twitter.com, you’d simply type a[href=’twitter.com’]

  5. * {

    The asterisk character by itself will select all elements on the page, which can be useful if you want to apply some common rules like zeroing margins and paddings. The asterisk can also be used to select all children of an element like so: div * {...}, but this isn’t very commonly used

  6. element-a element-b {

    Putting a space between two elements will select all instances of element-b on the page that are descendants of an element-a

  7. element-a, element-b {

    Adding a comma to separate elements rather than a space applied the same rules to both elements. It’s basically just shorthand for element-a { … } element-b { … }

  8. element-a > element-b {

    Putting a greater-than sign between two elements is similar to leaving a space in between, but now will only target instances of element-b that are children, or direct descendants, of element-a

  9. element-a ~ element-b {

    This selector will target all instances of element-b that are siblings (that is, they share the same parent) of an element-a

  10. element-a + element-b {

    The plus selector is similar to the tilde in that it also selects siblings, but now will only select instances of element-b that appear immediately after element-a

  11. element:before { , element:after {

    The :before and :after pseudo selectors can be used to modify the space immediately before or after your element, and have been used in a wide variety of applications, most notably the clearfix hack

Phew, that was a lot of CSS. Hopefully we’ve demystified the process for you. Creating websites isn’t magic--it’s just combining different CSS properties with the structure of your HTML in a specific way. Once you get the hang of it, you’ll be surprised the things you can do with pure CSS, like make an entire functioning tic-tac-toe game.
